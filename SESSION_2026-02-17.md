## Исправленные баги

### Bug 1 — Дедлок stdout/stderr в FFmpegService и WhisperService

**Файлы:** `ClipNotes/Services/FFmpegService.cs`, `ClipNotes/Services/WhisperService.cs`

**Причина:** Оба сервиса читали stdout и stderr **последовательно** через `.ReadToEndAsync()`.
FFmpeg пишет большой объём прогресса в stderr. Когда буфер pipe (64 KB) заполняется, а процесс
ждёт читателя stderr — он блокируется. Но в коде в этот момент всё ещё читается stdout.
Результат: взаимная блокировка (deadlock), приложение зависает навсегда.

**Исправление:** Чтение обоих потоков **конкурентно** через `Task.WhenAll`:

```csharp
process.Start();
// Read stdout and stderr concurrently to prevent pipe buffer deadlock
var stdoutTask = process.StandardOutput.ReadToEndAsync(ct);
var stderrTask = process.StandardError.ReadToEndAsync(ct);
await Task.WhenAll(stdoutTask, stderrTask);
await process.WaitForExitAsync(ct);
var stdout = stdoutTask.Result;
var stderr = stderrTask.Result;
```

### Bug 2 — Неполная модель ggml-large-v3-turbo.bin

**Симптом:** whisper-cli падал с ошибкой `expected 587, got 305 tensors`.

**Причина:** Файл `ggml-large-v3-turbo.bin` был загружен неполностью — 756 MB вместо 1549 MB (49%).
Первоначальный `download-model.ps1` проверял только **наличие файла**, не его размер.

**Исправление в download-model.ps1:**
```powershell
# HEAD-запрос для получения ожидаемого размера
$req = [System.Net.WebRequest]::Create($url)
$req.Method = "HEAD"
$resp = $req.GetResponse()
$remoteSize = $resp.ContentLength

if ($remoteSize -gt 0 -and $localSize -ge $remoteSize) {
    Write-Host "Файл полный, пропускаем загрузку"
    return
} else {
    Write-Host "Файл неполный, перезагружаем..."
    Remove-Item $outputPath -Force
}
```

**Решение:** Загрузил `ggml-base.bin` (148 MB) для немедленного тестирования.
Запустил перезагрузку `ggml-large-v3-turbo.bin` в фоне — файл загрузился полностью (1549 MB).

видеофайлом, получает длительность через ffprobe, переходит на вкладку "Экспорт".

```csharp
[RelayCommand]
private async Task LoadExistingVideoAsync()
{
    var dialog = new Microsoft.Win32.OpenFileDialog
    {
        Title = "Выберите видеофайл для генерации",
        Filter = "Видеофайлы (*.mkv;*.mp4;*.avi;*.mov;*.flv;*.wmv)|..."
    };
    if (dialog.ShowDialog() != true) return;
    var sessionDir = _sessionService.CreateSessionFolder(OutputRootDirectory);
    _currentSession = new SessionData { VideoFilePath = dialog.FileName, ... };
    var dur = await _ffmpeg.GetDurationAsync(dialog.FileName);
    ReviewReady = true;
    CurrentTab = 2; // перейти на вкладку Экспорт
}
```

### 2. Ручное добавление и удаление маркеров

**Файл:** `ClipNotes/ViewModels/MainViewModel.cs`

- `AddManualMarkerCommand` — парсит тайм-код `ЧЧ:ММ:СС`, добавляет маркер, увеличивает тайм-код на 30с
- `RemoveMarkerCommand` — удаляет маркер, переиндексирует оставшиеся

```csharp
[ObservableProperty] private string _manualMarkerTimecode = "00:00:30";
[ObservableProperty] private MarkerType _manualMarkerType = MarkerType.Note;

[RelayCommand]
private void AddManualMarker()
{
    if (!TimeSpan.TryParseExact(ManualMarkerTimecode, @"hh\:mm\:ss", null, out var ts))
        return;
    var marker = new Marker { Index = Markers.Count + 1, Timestamp = ts, Type = ManualMarkerType };
    Markers.Add(marker);
    ManualMarkerTimecode = (ts + TimeSpan.FromSeconds(30)).ToString(@"hh\:mm\:ss");
}
```
